REGLAS TÉCNICAS OBLIGATORIAS

Proyecto:
- React + Vite.
- Tailwind CSS.
- GSAP + @gsap/react + ScrollTrigger.
- Landing one-page.
- No crear varias páginas.

============================================================
REGLA MÁS IMPORTANTE: COPY CENTRALIZADO
============================================================

Todo el copy debe vivir únicamente en:

src/content/LandingCopy.js

Ningún componente .jsx puede tener copy hardcodeado.

Permitido en componentes:
- nombres de props.
- aria-labels si vienen desde LandingCopy.js o constantes.
- clases Tailwind.
- lógica visual.

No permitido en componentes:
- titulares escritos a mano.
- textos de botones escritos a mano.
- FAQs escritas a mano.
- testimonios escritos a mano.
- nombres de personas escritos a mano.

Los componentes deben consumir:
import { landingCopy } from "../content/LandingCopy";

o recibir props desde App.jsx.

============================================================
DEPENDENCIAS RECOMENDADAS
============================================================

Instalar:
- gsap
- @gsap/react
- lenis
- lucide-react

Comandos sugeridos:
npm install gsap @gsap/react lenis lucide-react

============================================================
TAILWIND
============================================================

Configurar:
- fontFamily.heading = Sora
- fontFamily.body = Manrope
- colores por variables CSS o tokens.
- sombras/glow para CTA.

index.css:
- Importar Google Fonts o usar link en index.html.
- Definir variables CSS.
- Definir base body.
- Crear clases reutilizables:
  - .section-padding
  - .premium-container
  - .glass-card
  - .orange-glow
  - .text-balance

============================================================
ACCESIBILIDAD
============================================================

- Botones y links con aria-label.
- Contraste suficiente.
- Focus-visible claro.
- FAQ navegable con teclado.
- No bloquear scroll nativo.
- Respetar prefers-reduced-motion.

============================================================
PERFORMANCE
============================================================

- Imágenes con loading="lazy" salvo hero.
- Imágenes con decoding="async".
- Video con poster.
- No cargar assets externos innecesarios.
- Evitar animaciones pesadas en mobile.
- Evitar re-render de carruseles innecesario.

============================================================
RESPONSIVE
============================================================

Desktop:
- Layout premium, cinematográfico, con scroll animations más fuertes.

Tablet:
- Mantener buena lectura.
- Reducir pinning si genera problemas.

Mobile:
- CTA sticky inferior después del hero.
- Animaciones simples.
- Cards verticales.
- Sin horizontal scroll difícil de usar.
- Texto legible.

============================================================
VALIDACIÓN FINAL
============================================================

Antes de terminar, Claude debe verificar:
1. npm run dev funciona.
2. npm run build funciona.
3. Todo el copy está en LandingCopy.js.
4. No hay copy hardcodeado en componentes .jsx.
5. Todos los CTA consumen landingCopy.cta.
6. La landing es one-page.
7. ScrollTrigger está registrado correctamente.
8. Las animaciones tienen cleanup.
9. Mobile está cuidado.
10. La estética se siente premium, ejecutiva y sofisticada.
